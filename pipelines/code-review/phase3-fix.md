# Fix Issues

Work through `docs/code-quality-findings.md` and fix each issue. Address
them in priority order. After each group of related changes, build and test
to confirm nothing is broken before moving on.

## Rules

- Fix one issue or one group of related issues at a time
- Build and test after each change (`cargo test`, `npm test`, etc.)
- If a fix breaks something, diagnose and resolve before continuing
- Update the findings doc as you go: mark items done or note why they were skipped
- Do not refactor beyond what the findings doc calls for
